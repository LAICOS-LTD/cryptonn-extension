> **CryptONN Loader является бесплатным и не требует лицензионного ключа.** Лицензирование осуществляется на этапе кодирования, а не на уровне загрузчика. Установите один раз на сервер — он автоматически обрабатывает все защищённые приложения.

## Что такое CryptONN?

CryptONN — это профессиональная платформа для защиты исходного кода PHP и управления программными лицензиями, созданная для независимых поставщиков программного обеспечения (ISV) и команд разработчиков, которые распространяют PHP-приложения коммерчески. Платформа преобразует исходные файлы PHP в непрозрачный зашифрованный бинарный формат, принципиально устойчивый к обратной разработке, декомпиляции и несанкционированному распространению — при полном сохранении производительности и совместимости со стандартной инфраструктурой PHP.

Система состоит из двух компонентов: **CryptONN Encoder** (настольное приложение, используемое разработчиком для защиты файлов PHP) и **CryptONN Loader** (данный репозиторий — единственный PHP-файл, устанавливаемый на сервере конечного клиента для прозрачного выполнения защищённых файлов).

## Решаемые проблемы

| Проблема | Решение CryptONN |
|---|---|
| **Кража исходного кода** | PHP-логика преобразуется в зашифрованный бинарный payload. Даже при полном доступе к файловой системе исходный код не может быть восстановлен. |
| **Несанкционированное развёртывание** | Каждый защищённый файл содержит встроенный идентификатор лицензии, проверяемый на стороне сервера. Файлы, скопированные на нелицензированные серверы, отказываются выполняться. |
| **Соблюдение условий лицензии** | Пробные периоды и даты истечения срока действия применяются на сервере лицензирования. Обход на стороне клиента невозможен. |
| **Многоарендная дистрибуция** | Одна и та же кодовая база может быть лицензирована множеству клиентов, каждый с уникальными условиями, ограничениями использования и ограничениями домена. |
| **Несанкционированное распространение** | Защищённые файлы привязаны к конкретным идентификаторам лицензий — они бесполезны без активной действующей лицензии. |

## Принцип работы

**При кодировании** (машина разработчика):
Файл PHP обрабатывается CryptONN Encoder. На выходе получается бинарный файл `.cryptonn`, содержащий зашифрованный payload и встроенный идентификатор лицензии. Исходный PHP-код отсутствует в выходных данных в любом виде.

**При выполнении** (сервер клиента):
1. PHP пытается выполнить файл `.cryptonn`
2. Загрузчик перехватывает выполнение через механизм `auto_prepend_file` PHP
3. Загрузчик считывает встроенный идентификатор лицензии из заголовка файла
4. Загрузчик обращается к API лицензирования CryptONN, предоставляя идентификатор лицензии и уникальный отпечаток, производный от сетевой идентичности сервера
5. API проверяет лицензию и возвращает ключ расшифровки, зашифрованный специально для данного сервера
6. Загрузчик расшифровывает PHP-payload полностью в памяти
7. Расшифрованный PHP-код выполняется нативно — никакие временные файлы с исходным кодом не сохраняются на диске

## Модель безопасности

| Свойство | Детали |
|---|---|
| **Ключи на сервере** | Отсутствуют. Никакие криптографические ключи не хранятся в файловой системе или среде. |
| **Привязка к серверу** | Каждый сервер имеет уникальный отпечаток, производный от его сетевой идентичности. Пакет расшифровки, действительный для одного сервера, криптографически бесполезен на любом другом. |
| **Взаимодействие с API** | Вся доставка ключей осуществляется по зашифрованным HTTPS-каналам. Ключи дополнительно шифруются с использованием специфического для сервера ключа обёртки перед передачей. |
| **Офлайн-устойчивость** | Трёхуровневый кэш (в процессе → файловый 24ч → период отсрочки 72ч) обеспечивает работу при временных сбоях API. |
| **Применение пробного режима** | Пробные лицензии имеют жёсткое истечение срока на стороне сервера. Период отсрочки не применяется — API должна подтвердить действительность пробных лицензий при каждом промахе кэша. |
| **Обнаружение вмешательства** | Усечённые, изменённые или повреждённые файлы `.cryptonn` обнаруживаются и отклоняются до любой попытки расшифровки. |
| **Нет открытого текста на диске** | Расшифрованный PHP-код никогда не записывается в постоянное место хранения. Временные файлы выполнения немедленно удаляются после использования. |

## Системные требования

| Компонент | Требование | Примечания |
|---|---|---|
| PHP | 7.2 – 8.5 | Все минорные версии поддерживаются |
| ext-sodium | Любая версия | Включён в PHP 7.2+ — отдельная установка не требуется |
| ext-openssl | Любая версия | Доступен по умолчанию практически во всех хостинговых средах |
| Исходящий HTTPS | Порт 443 | Необходим для вызовов API проверки лицензий |
| Диск (кэш) | ~10 КБ на активную лицензию | Временные файлы в системной временной директории |
| APCu (опционально) | Любая версия | Включает внутрипроцессное кэширование; значительно снижает задержку холодного старта |

## Установка

### Шаг 1 — Загрузка загрузчика

```bash
sudo mkdir -p /opt/cryptonn
sudo curl -fsSL https://raw.githubusercontent.com/LAICOS-LTD/cryptonn-extension/main/cryptonn-loader.php \
     -o /opt/cryptonn/cryptonn-loader.php
sudo chmod 644 /opt/cryptonn/cryptonn-loader.php
sudo chown root:root /opt/cryptonn/cryptonn-loader.php
```

### Шаг 2 — Настройка PHP (выберите среду)

**cPanel / EasyApache 4**
```bash
echo "auto_prepend_file = /opt/cryptonn/cryptonn-loader.php" \
  >> /opt/cpanel/ea-phpXX/root/etc/php.ini
/scripts/restartsrv_apache && /scripts/restartsrv_php_fpm
```

**Plesk / DirectAdmin — `.user.ini`**
```ini
auto_prepend_file = /opt/cryptonn/cryptonn-loader.php
```

**Выделенный сервер — PHP-FPM**
```ini
php_admin_value[auto_prepend_file] = /opt/cryptonn/cryptonn-loader.php
```
```bash
systemctl restart php8.2-fpm
```

**Apache — `.htaccess`**
```apache
php_value auto_prepend_file /opt/cryptonn/cryptonn-loader.php
```

### Шаг 3 — Проверка установки

Сохраните следующий код как `/tmp/cnn-verify.php` и выполните:

```php
<?php
echo defined('_CNN_MAGIC') ? "✅ CryptONN Loader: Активен\n" : "❌ CryptONN Loader: Не загружен\n";
echo "Версия PHP  : " . PHP_VERSION . "\n";
echo "ext-sodium  : " . (extension_loaded('sodium')  ? "✅" : "❌ ОТСУТСТВУЕТ") . "\n";
echo "ext-openssl : " . (extension_loaded('openssl') ? "✅" : "❌ ОТСУТСТВУЕТ") . "\n";
echo "APCu        : " . (function_exists('apcu_store') ? "✅ Доступен" : "— Недоступен (опционально)") . "\n";
```

```bash
php /tmp/cnn-verify.php
```

## Устранение неполадок

### `CryptONN Loader requires ext-sodium`
**Причина:** Расширение `sodium` не включено для активной версии PHP.

```bash
# cPanel / EasyApache 4
/scripts/install_ea_metapackage ea-php82-php-sodium

# AlmaLinux / RHEL / CentOS 8+
dnf install php-sodium

# Ubuntu / Debian
apt-get install php8.2-sodium

# Проверка
php -m | grep sodium
```

---

### `CryptONN Loader requires ext-openssl`
**Причина:** Расширение `openssl` не включено.

Включите `extension=openssl` в соответствующем `php.ini` или установите пакет `php-openssl` через менеджер пакетов.

---

### `Master key could not be retrieved`
**Причина:** Загрузчик не может подключиться к API лицензирования CryptONN. Это может быть вызвано правилом брандмауэра, блокирующим исходящий HTTPS, ошибкой разрешения DNS или временной проблемой сети.

```bash
curl -sv --max-time 10 https://api.laicos.com.tr/health
```

**Решения:**
- Убедитесь, что исходящий TCP-порт 443 разрешён с сервера
- Проверьте, что сервер может разрешать внешние DNS-имена
- При использовании исходящего прокси настройте переменную среды `CRYPTONN_API_URL`

---

### `Invalid magic bytes`
**Причина:** Файл не является допустимым CryptONN-файлом, или он был повреждён при передаче (например, передан в текстовом режиме вместо бинарного).

**Решение:** Повторно передайте файл `.cryptonn` в бинарном режиме. Не открывайте и не редактируйте файл текстовым редактором.

---

### `Incomplete header`
**Причина:** Файл `.cryptonn` обрезан — он не был передан полностью.

**Решение:** Повторно передайте файл. Проверьте доступное дисковое пространство на источнике и получателе.

---

### `Decryption failed`
**Причина:** Ключ расшифровки, возвращённый API, не соответствует параметрам шифрования файла.

**Решение:** Уточните у поставщика программного обеспечения, что при кодировании файла использовался правильный идентификатор лицензии.

---

### `Temporary file could not be written`
**Причина:** PHP-процесс не имеет прав на запись в системный временный каталог.

**Решение:** Убедитесь, что пользователь веб-сервера имеет доступ на запись к `sys_get_temp_dir()` (обычно `/tmp`). Проверьте политики SELinux или AppArmor на защищённых системах.

## Часто задаваемые вопросы

**В: Загрузчик бесплатный?**  
О: Да. CryptONN Loader является бесплатным и с открытым исходным кодом. Нет лицензионного ключа, подписки или платы за установку на любое количество серверов.

**В: Работает ли с PHP OPcache?**  
О: Да. OPcache работает с байткодом PHP после того, как загрузчик расшифровал и выполнил код. Взаимодействие полностью прозрачно и корректно.

**В: Может ли одна установка загрузчика обслуживать несколько приложений?**  
О: Да. Единственная установка загрузчика обрабатывает все файлы `.cryptonn` на сервере во всех приложениях и версиях PHP, которые ссылаются на него через `auto_prepend_file`.

**В: Что происходит при сбое API?**  
О: Непробные лицензии продолжают нормально работать до 72 часов, используя файловый кэш. Пробные лицензии требуют успешного ответа API при каждом промахе кэша — они не пользуются периодом отсрочки.

**В: Представляют ли кэшированные данные угрозу безопасности?**  
О: Нет. Кэшированный пакет зашифрован ключом, производным от уникального отпечатка сервера. Его невозможно расшифровать на другой машине, и он не содержит необработанный ключ расшифровки в какой-либо используемой форме.

## Удаление загрузчика

1. Удалите директиву `auto_prepend_file` из `php.ini`, `.user.ini` или `.htaccess`
2. Перезапустите PHP-FPM или Apache
3. Удалите каталог загрузчика:
```bash
rm -rf /opt/cryptonn
```

Это не затрагивает файлы `.cryptonn` — они просто перестают выполняться до повторной установки загрузчика.
