> **CryptONN Loader مجاني تمامًا ولا يتطلب أي مفتاح ترخيص.** تتم إدارة الترخيص في مرحلة التشفير، وليس على مستوى المُحمِّل. ثبِّته مرة واحدة على السيرفر ليدير تلقائيًا جميع التطبيقات المحمية.

## ما هو CryptONN؟

CryptONN منصة احترافية لحماية الكود المصدري PHP وإدارة تراخيص البرمجيات، مصمَّمة لموردي البرمجيات المستقلين (ISV) وفرق التطوير التي تُوزِّع تطبيقات PHP تجاريًا. تُحوِّل المنصة ملفات PHP المصدرية إلى صيغة ثنائية مشفَّرة تقاوم بشكل جوهري الهندسة العكسية وإلغاء التصريف وإعادة التوزيع غير المصرَّح به — مع الحفاظ على الأداء الكامل والتوافق مع بنية PHP القياسية.

يتكوَّن النظام من مكوِّنَين: **CryptONN Encoder** (تطبيق سطح مكتب يستخدمه المطوِّر لحماية ملفات PHP) و**CryptONN Loader** (هذا المستودع — ملف PHP واحد يُثبَّت على سيرفر العميل النهائي لتنفيذ الملفات المحمية بشفافية).

## المشكلات التي يحلها

| المشكلة | حل CryptONN |
|---|---|
| **سرقة الكود المصدري** | تُحوَّل منطق PHP إلى حمولة ثنائية مشفَّرة. حتى مع الوصول الكامل لنظام الملفات، لا يمكن إعادة بناء الكود الأصلي. |
| **النشر غير المصرَّح به** | يحمل كل ملف محمي معرِّف ترخيص مُضمَّنًا يُتحقَّق منه على جانب السيرفر. الملفات المنسوخة على سيرفرات غير مرخَّصة ترفض التنفيذ. |
| **تطبيق شروط الترخيص** | تُطبَّق فترات التجريب وتواريخ الانتهاء على سيرفر الترخيص. لا توجد فحوصات على جانب العميل يمكن تجاوزها. |
| **التوزيع متعدد المستأجرين** | يمكن ترخيص قاعدة الكود ذاتها لعملاء متعددين، لكل منهم شروط وحدود استخدام وقيود نطاق فريدة. |
| **إعادة التوزيع غير المصرَّح به** | الملفات المحمية مرتبطة بمعرِّفات ترخيص محددة — لا قيمة لها بدون ترخيص نشط وصالح. |

## كيفية العمل

**عند التشفير** (جهاز المطوِّر):
يُعالج ملف PHP بواسطة CryptONN Encoder. الناتج ملف ثنائي `.cryptonn` يحتوي على حمولة مشفَّرة ومعرِّف ترخيص مُضمَّن. الكود المصدري الأصلي غير موجود في الناتج بأي شكل.

**عند وقت التشغيل** (سيرفر العميل):
1. يحاول PHP تنفيذ ملف `.cryptonn`
2. يعترض المُحمِّل التنفيذ عبر آلية `auto_prepend_file` في PHP
3. يقرأ المُحمِّل معرِّف الترخيص المُضمَّن من رأس الملف
4. يتصل المُحمِّل بواجهة API ترخيص CryptONN، مُقدِّمًا معرِّف الترخيص وبصمة فريدة مشتقَّة من هوية شبكة السيرفر
5. تُتحقَّق API من الترخيص وتُعيد مفتاح فك التشفير مشفَّرًا خصيصًا لهذا السيرفر
6. يفك المُحمِّل تشفير حمولة PHP بالكامل في الذاكرة
7. ينفَّذ كود PHP المفكوك تشفيره بشكل أصلي — لا يُحتفظ بأي ملفات مؤقتة تحتوي على الكود المصدري على القرص

## نموذج الأمان

| الخاصية | التفاصيل |
|---|---|
| **المفاتيح المخزَّنة على السيرفر** | لا يوجد. لا تُخزَّن مفاتيح تشفيرية في نظام الملفات أو البيئة. |
| **الربط بالسيرفر** | لكل سيرفر بصمة فريدة مشتقَّة من هويته الشبكية. حزمة فك التشفير الصالحة لسيرفر واحد عديمة الجدوى تشفيريًا على أي سيرفر آخر. |
| **التواصل مع API** | يجري تسليم جميع المفاتيح عبر قنوات HTTPS مشفَّرة. تُشفَّر المفاتيح إضافيًا بمفتاح تغليف خاص بالسيرفر قبل الإرسال. |
| **التسامح مع قطع الاتصال** | تضمن ذاكرة تخزين مؤقت ثلاثية الطبقات (في العملية ← ملف 24 ساعة ← فترة سماح 72 ساعة) التشغيل أثناء انقطاعات API المؤقتة. |
| **تطبيق وضع التجربة** | تفرض تراخيص التجربة انتهاء صلاحية صارمة على السيرفر. لا تُطبَّق فترة سماح بلا اتصال — يجب أن تُؤكِّد API صلاحية تراخيص التجربة عند كل إخفاق في ذاكرة التخزين المؤقت. |
| **اكتشاف التلاعب** | تُكتشَف ملفات `.cryptonn` المقطوعة أو المعدَّلة أو التالفة وترفض قبل أي محاولة فك تشفير. |
| **لا نص واضح على القرص** | لا يُكتب كود PHP المفكوك تشفيره أبدًا في موقع دائم. تُحذف ملفات التنفيذ المؤقتة فور الانتهاء منها. |

## متطلبات النظام

| المكوِّن | المتطلب | ملاحظات |
|---|---|---|

## التثبيت

### الخطوة الأولى — تنزيل المُحمِّل

```bash
sudo mkdir -p /opt/cryptonn
sudo curl -fsSL https://raw.githubusercontent.com/LAICOS-LTD/cryptonn-loader/main/cryptonn-loader.php \
     -o /opt/cryptonn/cryptonn-loader.php
sudo chmod 644 /opt/cryptonn/cryptonn-loader.php
sudo chown root:root /opt/cryptonn/cryptonn-loader.php
```

### الخطوة الثانية — إعداد PHP (اختر بيئتك)

**cPanel / EasyApache 4**
```bash
echo "auto_prepend_file = /opt/cryptonn/cryptonn-loader.php" \
  >> /opt/cpanel/ea-phpXX/root/etc/php.ini
/scripts/restartsrv_apache && /scripts/restartsrv_php_fpm
```

**Plesk / DirectAdmin — `.user.ini`**
```ini
auto_prepend_file = /opt/cryptonn/cryptonn-loader.php
```

**السيرفر المخصص — PHP-FPM**
```ini
php_admin_value[auto_prepend_file] = /opt/cryptonn/cryptonn-loader.php
```
```bash
systemctl restart php8.2-fpm
```

**Apache — `.htaccess`**
```apache
php_value auto_prepend_file /opt/cryptonn/cryptonn-loader.php
```

### الخطوة الثالثة — التحقق من التثبيت

احفظ الكود التالي بالاسم `/tmp/cnn-verify.php` ثم نفِّذه:

```php
<?php
echo defined('_CNN_MAGIC') ? "✅ CryptONN Loader: نشط\n" : "❌ CryptONN Loader: غير محمَّل\n";
echo "إصدار PHP   : " . PHP_VERSION . "\n";
echo "ext-sodium  : " . (extension_loaded('sodium')  ? "✅" : "❌ مفقود") . "\n";
echo "ext-openssl : " . (extension_loaded('openssl') ? "✅" : "❌ مفقود") . "\n";
echo "APCu        : " . (function_exists('apcu_store') ? "✅ متاح" : "— غير متاح (اختياري)") . "\n";
```

```bash
php /tmp/cnn-verify.php
```

## استكشاف الأخطاء وإصلاحها

### `CryptONN Loader requires ext-sodium`
**السبب:** إضافة `sodium` غير مُفعَّلة لإصدار PHP النشط.

```bash
# cPanel / EasyApache 4
/scripts/install_ea_metapackage ea-php82-php-sodium

# AlmaLinux / RHEL / CentOS 8+
dnf install php-sodium

# Ubuntu / Debian
apt-get install php8.2-sodium

# التحقق
php -m | grep sodium
```

---

### `CryptONN Loader requires ext-openssl`
**السبب:** إضافة `openssl` غير مُفعَّلة.

فعِّل `extension=openssl` في `php.ini` المناسب، أو ثبِّت حزمة `php-openssl` عبر مدير الحزم.

---

### `Master key could not be retrieved`
**السبب:** لا يستطيع المُحمِّل الوصول إلى API ترخيص CryptONN. قد يكون ذلك بسبب قاعدة جدار حماية تحجب HTTPS الصادر، أو فشل في استيفاء DNS، أو مشكلة شبكة مؤقتة.

```bash
curl -sv --max-time 10 https://api.laicos.com.tr/health
```

**الحلول:**
- تأكَّد من السماح بمنفذ TCP 443 الصادر من السيرفر
- تحقَّق من قدرة السيرفر على استيفاء أسماء DNS الخارجية
- إذا كنت خلف بروكسي صادر، عيِّن متغير البيئة `CRYPTONN_API_URL`

---

### `Invalid magic bytes`
**السبب:** الملف ليس ملف CryptONN صالحًا، أو تلف أثناء النقل (مثلاً نُقل في وضع النص بدلاً من الثنائي).

**الحل:** أعِد نقل ملف `.cryptonn` في الوضع الثنائي. لا تفتح الملف أو تعدِّله بمحرر نصوص.

---

### `Incomplete header`
**السبب:** ملف `.cryptonn` مقطوع — لم يُنقل بالكامل.

**الحل:** أعِد نقل الملف. تحقَّق من المساحة المتاحة على القرص في المصدر والوجهة.

---

### `Decryption failed`
**السبب:** مفتاح فك التشفير الذي أعادته API لا يطابق معاملات التشفير للملف.

**الحل:** تأكَّد مع مورد البرنامج أن معرِّف الترخيص الصحيح استُخدم عند تشفير الملف.

---

### `Temporary file could not be written`
**السبب:** عملية PHP لا تملك صلاحية الكتابة في دليل temp النظام.

**الحل:** تأكَّد من أن مستخدم خادم الويب يملك صلاحية الكتابة في `sys_get_temp_dir()` (عادةً `/tmp`). راجع سياسات SELinux أو AppArmor على الأنظمة المُصلَّبة.

## الأداء

| طبقة التخزين المؤقت | زمن الاستجابة النموذجي | المدة |
|---|---|---|
| داخل العملية (APCu) | < 0.1 مللي ثانية | ساعة واحدة |
| تخزين الملف المؤقت | < 0.5 مللي ثانية | 24 ساعة |
| استدعاء API (بارد) | 50 – 200 مللي ثانية | عند إخفاق التخزين المؤقت |
| فترة السماح | < 0.5 مللي ثانية | حتى 72 ساعة (غير تجريبي) |

يُستخدَم APCu تلقائيًا عند توفُّره. لا يلزم أي إعداد إضافي.

## الأسئلة الشائعة

**س: هل المُحمِّل مجاني؟**  
ج: نعم. CryptONN Loader مجاني ومفتوح المصدر. لا يوجد مفتاح ترخيص أو اشتراك أو رسوم مرتبطة بتثبيته على أي عدد من السيرفرات.

**س: هل يعمل مع PHP OPcache؟**  
ج: نعم. يعمل OPcache على بايت كود PHP بعد أن يفك المُحمِّل تشفير الكود وينفِّذه. التفاعل شفاف تمامًا وصحيح.

**س: هل يمكن لتثبيت واحد للمُحمِّل خدمة تطبيقات متعددة؟**  
ج: نعم. يعالج تثبيت واحد للمُحمِّل جميع ملفات `.cryptonn` على السيرفر، عبر جميع التطبيقات وإصدارات PHP التي تُشير إليه عبر `auto_prepend_file`.

**س: ماذا يحدث أثناء انقطاع API؟**  
ج: تستمر التراخيص غير التجريبية في العمل بشكل طبيعي لمدة تصل إلى 72 ساعة باستخدام تخزين الملف المؤقت. تراخيص التجربة تتطلب استجابة API ناجحة عند كل إخفاق في التخزين المؤقت — لا تستفيد من فترة السماح.

**س: هل البيانات المخزَّنة مؤقتًا تُشكِّل خطرًا أمنيًا؟**  
ج: لا. الحزمة المخزَّنة مؤقتًا مشفَّرة بمفتاح مشتقّ من البصمة الفريدة للسيرفر. لا يمكن فك تشفيرها على أي جهاز آخر، ولا تحتوي على مفتاح فك التشفير الخام بأي شكل قابل للاستخدام.

## إزالة المُحمِّل

1. أزِل توجيه `auto_prepend_file` من `php.ini` أو `.user.ini` أو `.htaccess`
2. أعِد تشغيل PHP-FPM أو Apache
3. احذف دليل المُحمِّل:
```bash
rm -rf /opt/cryptonn
```

لا يؤثر ذلك على أي ملفات `.cryptonn` — تتوقف فقط عن التنفيذ حتى إعادة تثبيت المُحمِّل.
